## ✨ מה מייחד את הפרויקט?
- **שימוש בלמידה עמוקה**

אני בחרתי להשתמש ברשותות ניורונים. ב **RNN** אם להיות מדויק. זהוי סוג רשת שמתאימה למידע סידרתי.
יש הרבה סיבות למה חשבתי שזו תיהיה אפשרות יותר טובה: הרשת יכולה ללמוד קשרים בין קבוצות שונות, להשליך מהעבר על העתיד וללמוד אוטומטית את הפונקציה המתאימה לכל קבוצה.
למעשה רשתות ניורונים יכולות ללמוד כל פונקציה באופן כללי.

- **תוכנה איכותית**

השתדלתי כמה שאני יכול שהתוכנית תיהיה איכותית, הן מבחינת קוד והן מבחינת מדעי הנתונים. היתרון של עבודה זו בעיני, יותר מהשימוש בטכנולוגיה שאני ראיתי לנכון, זו הפשטות. השתדלתי שהתוכנית תיהיה אלגנטית וקלה לשימוש. חשוב היה לי לאזן בין דיוק בתוצאה הסופית לבין הפשטות של העבודה. עבודה מסורבלת שורפת זמן, נתקעת בקלות ובסופו של דבר אינה ברת קיימא.

- טעות מוחלטת

**משתני האימון**

ישנה האפשרות לבחור את ערכי המשתנים המשפעים על האימון.
אפשר לראות את כל המשתנים האלו יחד עם שמות העמודות ויתר הקבועים בתחילת התוכנית.

- learning rate, קצב האימון.
- batch size, על כמה שנים מתאמנים בכל חזרה.
- hidden size, כמה ניורונים בשכבה של תא ה LSTM.
- epochs, כמה חזרות בלולאת אימון.
- input path, הנתיב לקלט
- output path, הנתיב לפלט
- model path, הנתיב לקובץ ששומר את משתני הרשת המאומנת.
- train, האם לאמן את הרשת מחדש או רק לשלוף אותה לחיזוי.
- seed, כמה חודשים מתחילת השנה הנחזת לגלות לרשת בחיזוי.

**חשוב:**

אפשר לבחור לפי איזה גורם לחלק את המידע. לי עבד הכי טוב קידומות GL. אפשר גם לבחור כמה והתוכנית תחלק לכל צירוף אפשרי במידע ותסנן צירופים כמעט ריקים.

בשלב החיזוי אפשר לבחור כפי שכתבתי, כמה חודשים משנת המטרה לגלות למודל. בפועל, אם משתמשים בעבודה לחיוזוי שנה שותפת עדיף לגלות עד החודש העדכני ביותר. ואם רק בודקים את התוכנית על שנים אחרות אז אפשר לראות איך הרשת מתמודדת עם כמות חודשים מקדימים שונה.

כדאי לשמור את תוצאות כל ניסוי בתקיה משלו. קובץ המודל השמור צריך להיות בסיומת .pt.
